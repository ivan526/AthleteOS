# AthleteOS 使用说明

## 🚀 快速启动

### 方式一：一键启动（推荐）
直接双击运行 `start.bat` 脚本，会自动：
1. 停止所有已运行的相关服务
2. 启动后端服务（端口 3007）
3. 启动前端服务（端口 3008）
4. 自动打开两个命令窗口分别显示前后端日志

### 方式二：多功能管理脚本
在项目根目录打开命令提示符，运行：
```bash
# 查看帮助
app

# 启动所有服务
app start

# 停止所有服务
app stop

# 重启所有服务
app restart

# 查看服务状态
app status

# 仅启动后端
app backend

# 仅启动前端
app frontend
```

### 方式三：手动启动
**启动后端：**
```bash
cd backend
npm run start:dev
```

**启动前端：**
```bash
cd frontend
npm run dev
```

## 🌐 访问地址
- 前端应用：http://localhost:3008
- 后端API：http://localhost:3007
- API文档：http://localhost:3007/api

## 🛠️ 常用命令

### 停止服务
- 直接关闭前后端的命令窗口即可
- 或者运行 `stop.bat` 一键停止所有服务
- 或者运行 `app stop`

### 清理缓存
如果遇到前端模块加载错误，运行：
```bash
cd frontend
rm -rf node_modules/.vite
```
然后重新启动前端服务。

## 📱 功能说明

### 核心页面
1. **今日训练** - 查看每日训练建议、训练能力评分、训练风险
2. **历史记录** - 查看历史训练记录，支持周/月视图切换
3. **每周复盘** - 查看周训练统计和分析
4. **设置** - 管理个人信息、偏好设置、同步Intervals.icu数据

### 主要功能
- ✅ Intervals.icu 数据同步
- ✅ 训练能力智能评分（0-100分）
- ✅ 训练风险评估
- ✅ 个性化训练建议生成
- ✅ 训练计划动态调整（根据用户反馈）
- ✅ 训练计时功能
- ✅ 历史训练记录查询
- ✅ 周训练复盘分析

## 🔧 技术栈
- **后端：** NestJS 11 + TypeScript + Prisma + SQLite
- **前端：** React 19 + TypeScript + Vite + Tailwind CSS
- **数据来源：** Intervals.icu API

## ⚠️ 注意事项
1. 启动前确保端口 3007 和 3008 未被其他程序占用
2. 两个命令窗口不要关闭，关闭即停止服务
3. 首次启动需要等待依赖安装和编译完成
4. 如果前端页面显示空白，按 `Ctrl + Shift + R` 强制刷新清除浏览器缓存

## 📝 常见问题

### Q: 启动时提示端口被占用？
A: 运行 `stop.bat` 或者 `app stop` 停止所有服务后重新启动。

### Q: 前端页面显示模块导入错误？
A: 清理Vite缓存：`cd frontend && rm -rf node_modules/.vite`，然后重新启动前端。

### Q: 如何同步Intervals.icu数据？
A: 进入「设置」页面，点击「同步Intervals.icu数据」按钮。

### Q: API返回错误怎么办？
A: 系统内置降级机制，API不可用时会自动使用模拟数据，不影响功能体验。
