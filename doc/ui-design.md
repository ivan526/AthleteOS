AthleteOS MVP UI 设计文档
版本：MVP UI Design v1.1
适用阶段：第一阶段 MVP
目标用户：中国耐力运动用户
主要数据源：Intervals.icu API
开发工具：opencode
视觉方向：中文优先、淡绿色主色、清新专业、低压健康感

1. 设计目标
AthleteOS 第一阶段界面的核心目标是：
用户每天打开产品后，10 秒内知道今天该怎么练、为什么这样练、如果状态变化应该怎么调整。
第一阶段不做复杂看板，不做社交，不做排行榜，不做大模型自由聊天。
界面重点是：
今日训练建议清晰可执行
训练能力状态一眼可见
解释足够简单，不堆复杂指标
用户可以快速反馈今日状态
系统可以基于反馈调整训练
数据来源明确来自 Intervals.icu API
风险表达克制，不制造焦虑

2. 设计原则
2.1 中文优先
目标客户是中国用户，因此界面文案必须中文优先。
保留英文的场景：
Intervals.icu
TSS
CTL
ATL
Form
ACWR
HRV
API
页面命名统一中文化：
底部导航统一为：
今日
历史
复盘
设置

2.2 淡绿色主色调
整体视觉从科技蓝调整为清新淡绿色。
设计关键词：
清新
健康
专业
克制
安全感
低压
可信赖
不要做成强刺激运动 App，也不要做成复杂金融数据看板。

2.3 数据源明确
MVP 阶段唯一外部训练数据源是：
Intervals.icu API
界面中应明确表达：
数据来源：Intervals.icu
不要在 MVP 界面中暗示已经直接接入：
Garmin API
Apple Health API
Strava API
Coros API
Whoop API
Oura API
如果某些恢复数据暂未接入，应明确写：
暂未接入
不要写成“同步中”，避免误导用户。

2.4 不做复杂 Dashboard
首页不叫 Dashboard。
首页叫：
今日训练
首页只解决 4 个问题：
今天状态如何？
今天练什么？
为什么这样练？
今天情况变化了怎么调整？

14.7 UI 验收标准
完成后应满足：
所有主界面中文优先。
主色调为淡绿色。
今日训练页一屏内能看到今日训练能力和今日建议。
今日训练页有“今天情况有变化？”反馈入口。
调整弹窗能展示调整前后对比。
决策依据页能展示专业指标，但不干扰首页。

每周复盘页能展示完成率、周负荷、恢复趋势、训练风险和下周建议。
不出现“Dashboard”字样。
不出现“受伤概率 xx%”字样。

16. 第一阶段成功标准
第一阶段 UI 成功的标志是：
用户打开“今日训练”后，不需要理解复杂指标，也能快速知道：
今天状态如何，
今天练什么，
为什么这样安排，
如果今天太累或没时间，如何调整。
界面必须让用户感受到：
这个产品不是单纯记录训练，
而是真的在帮我做训练决策。
